Design: SAP-Feature Architektur

Überblick über alle Änderungen — bitte kurz durchschauen

1. Datenschicht — Firestore Userdokument

users/{uid} — neue Felder
isSapUser: true / false ← wird beim Register/Login gesetzt
theme: "default" | "sap" | "minimalist"
sapIntensity: "badge" | "full" | "subtle" ← nur relevant wenn theme = "sap"

2. SAP-Erkennung — auth.js

ensureUserDoc() — Erweiterung
if (email.endsWith('@sap.com')) {
  isSapUser = true
  theme = 'sap' ← Standard für SAP-Nutzer
  sapIntensity = 'full'
}

3. Theme-Anwendung — main.js + style.css

CSS-Klassen auf <body>
Standard: <body> (keine extra Klasse)
SAP full: <body class="theme-sap theme-sap-full">
SAP subtle: <body class="theme-sap theme-sap-subtle">
SAP badge: <body class="theme-sap theme-sap-badge">
Minimalist: <body class="theme-minimalist">

4. Was sich visuell ändert (pro Theme)

Theme Farben Logo/Header Badge
Standard Blau (#0064e0) DHBW Lernportal —
SAP Full SAP Gold (#F0AB00) + Dunkelblau SAP Logo + "Lernportal" SAP Intern 🔵
SAP Subtle Akzente in SAP Gold DHBW Lernportal SAP Intern 🔵
SAP Badge Keine Änderung DHBW Lernportal SAP Intern 🔵
Minimalist Grau/Weiß, kein Primärblau DHBW Lernportal —

5. Neue Seite: SAP-Bereich

Route: #sap — nur für isSapUser = true sichtbar
• Neuer Menüpunkt im Dashboard-Nav (nur für SAP-Nutzer)
• Seite zeigt Platzhalter-Karten ("Inhalte folgen demnächst")
• Nicht-SAP-Nutzer die URL direkt aufrufen → Redirect zu Dashboard

6. Willkommens-Toast

Beim ersten SAP-Login — einmalig
• localStorage-Flag sap_welcomed verhindert Wiederholung
• Toast erscheint oben auf dem Dashboard: "Willkommen, SAP-Mitarbeiter! Dein exklusiver Bereich ist jetzt verfügbar."
• Verschwindet nach 4 Sekunden automatisch

7. Account-Seite — Theme-Einstellungen

profile.js — neuer Abschnitt "Erscheinungsbild"
• Gesamt-Theme wählen: Standard / SAP / Minimalist
• Wenn SAP: Intensität wählen: Badge / Subtil / Vollständig
• SAP-Optionen nur sichtbar wenn isSapUser = true
• Änderung wird sofort angewendet + in Firestore gespeichert
